1.Паттерн Abstract Factory( Створює структури даних різної спеціалізації );
2.Паттерн Decorator №1( Перетворює звичайний список в список для обчислення часу виконання операцій );
3.Паттерн Decorator №2( Перетворює звичайне Splay дерево в Splay дерево для обчислення часу виконання операцій );
4.Паттерн Decorator №3( Перетворює звичайне BPlus дерево в BPlus дерево для обчислення часу виконання операцій );
5.Паттерн Decorator №4( Перетворює звичайну Hash-таблицю в Hash-таблицю для обчислення часу виконання операцій );
6.Паттерн Adapter( Перетворює звичайний список в масив для використання в інших операціях );
7.Паттерн Strategy( Стратегія для найбільш оптимального сортування списку і масиву. 
Якщо розмір списку < 5, то використовуєм SelectionSort, якщо > 5 i < 15 - HeapSort, 
якщо > 15 i < 50 - MergeSort, якщо > 50 - BucketSort );
8.Паттерн Iterator №1( Потрібний для обходу списку );
9.Паттерн Iterator №2( Потрібний для обходу Hash-таблиці );
10.Паттерн Iterator №3( Потрібний для обходу Splay-дерева );
11.Паттерн Iterator №4( Потрібний для обходу B+-дерева );
